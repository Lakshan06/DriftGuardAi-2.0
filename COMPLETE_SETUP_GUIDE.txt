STEP 1: Verify Python Version
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Required: Python 3.8+
Recommended: Python 3.11 or 3.12

If you have Python 3.7 or older, upgrade:
  Windows: Download from python.org
  macOS: brew install python@3.12
  Linux: apt-get install python3.12


STEP 2: Navigate to Backend Directory
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

cd backend

Expected: You should be in C:\DriftGuardAI2.0\DriftGuardAi-2.0\backend


STEP 3: Verify Requirements Installation
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

pip list | grep -E "fastapi|pydantic|sqlalchemy"

Should show:
  fastapi                   0.109.0
  pydantic                  2.5.3
  pydantic-settings         2.1.0
  sqlalchemy                2.0.25

If any are missing, run:

STEP 4: Create Environment Configuration
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Copy the example .env file:
  cp .env.example .env

Edit .env and set:

  DATABASE_URL=postgresql://user:password@localhost:5432/driftguard_db
  
  # Replace with:
  # - user: your PostgreSQL username (default: postgres)
  # - password: your PostgreSQL password
  # - localhost: your database host
  # - driftguard_db: database name

  SECRET_KEY=<generate-32-random-characters>
  
  # Generate with:
  # python -c "import secrets; print(secrets.token_urlsafe(32))"
  
  ALGORITHM=HS256
  ACCESS_TOKEN_EXPIRE_MINUTES=30
  DRIFT_WINDOW_SIZE=100
  PSI_THRESHOLD=0.25
  KS_THRESHOLD=0.2
  FAIRNESS_THRESHOLD=0.1


STEP 5: Set Up PostgreSQL Database
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Verify PostgreSQL is running:

Windows:
  Services > PostgreSQL > Start (or Services app)

macOS:
  brew services start postgresql

Linux:
  sudo systemctl start postgresql

Create database:

Verify:

================================================================================
TROUBLESHOOTING
================================================================================

Problem 1: "ModuleNotFoundError: No module named 'fastapi'"
───────────────────────────────────────────────────────────

Solution:
  1. Ensure you're in backend directory: cd backend
  2. Reinstall requirements: pip install -r requirements.txt
  3. Verify installation: pip list | grep fastapi
  4. Try again: uvicorn app.main:app --reload


Problem 2: "ERROR: could not translate host name "localhost""
─────────────────────────────────────────────────────────────

This is a PostgreSQL connection error.

Solution:
  1. Verify PostgreSQL is running (see Step 5)
  2. Check DATABASE_URL in .env is correct
  3. Make sure database exists: createdb -U postgres driftguard_db
  4. Test connection: psql -U postgres -d driftguard_db


Problem 3: "Port 8000 already in use"
──────────────────────────────────────

Solution:
  Use a different port:
    uvicorn app.main:app --reload --port 8001

  Or kill existing process:
    lsof -i :8000  (find PID)
    kill -9 PID


Problem 4: "npm: command not found"
───────────────────────────────────

Solution:
  1. Install Node.js from nodejs.org
  2. Verify installation: node --version
  3. Try again: npm install && npm run dev


Problem 5: Email validation error on registration
──────────────────────────────────────────────────

Solution:
  1. Email must be valid format: user@example.com
  2. Cannot use: user@localhost
  3. Try: testuser@example.com


Problem 6: "Authentication failed" on API calls
───────────────────────────────────────────────

Solution:
  1. Make sure to login first (see Test 2)
  2. Copy the JWT token from response
  3. Add to headers: Authorization: Bearer YOUR_TOKEN


Problem 7: Frontend shows "Cannot GET /"
──────────────────────────────────────

Solution:
  1. Check frontend is running: npm run dev
  2. Try accessing: http://localhost:5173 (not 5174)
  3. Check console for errors
  4. Reload page
